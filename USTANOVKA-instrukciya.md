# Как обновить репозиторий

## Что в архиве

```
index.html          ← ЗАМЕНИТЬ существующий
styles.css          ← ЗАМЕНИТЬ существующий
README.md           ← ЗАМЕНИТЬ существующий
checker.js          ← новый файл
tabs.js             ← новый файл
spisok.json         ← новый файл
sobrat_spisok.py    ← новый файл
examples/
  tovary_primer.xlsx  ← новый файл
```

**Файла `app.js` в архиве нет — он не менялся.** Ваш существующий `app.js`
остаётся на месте нетронутым.

## Порядок действий

### Через веб-интерфейс GitHub

1. Распакуйте архив.
2. Откройте репозиторий, нажмите **Add file → Upload files**.
3. Перетащите в окно все файлы из архива **кроме папки `examples`**:
   `index.html`, `styles.css`, `README.md`, `checker.js`, `tabs.js`,
   `spisok.json`, `sobrat_spisok.py`.
4. GitHub предупредит, что три файла будут заменены — это нормально,
   так и задумано.
5. Внизу страницы нажмите **Commit changes**.
6. Отдельно загрузите `tovary_primer.xlsx` в папку `examples`:
   откройте папку `examples` в репозитории → **Add file → Upload files**.

### Через командную строку

```bash
cd путь/к/репозиторию
git pull

# скопировать файлы из архива поверх (кроме app.js — его в архиве нет)
cp -r путь/к/распакованному/архиву/* .

git add .
git commit -m "Добавлен раздел проверки ограничений 311, 312, 313"
git push
```

## Что проверить после обновления

GitHub Pages пересобирается за одну-две минуты. Откройте страницу и убедитесь:

1. **Первый раздел работает как раньше.** Загрузите `invoice_primer.xlsx`
   и `packing_list_primer.xlsx` — таблица должна собраться, дубликаты
   объединиться, расхождение по прокладке подсветиться жёлтым.
2. **Вкладки переключаются.** Наверху страницы два корешка: «01 Таблица
   товаров» и «02 Проверка ограничений».
3. **Второй раздел получает перечни.** Перейдите на вторую вкладку,
   нажмите «Обновить перечни с alta.ru». Под кнопкой должна появиться
   строка с числом позиций по каждому постановлению.
4. **Проверка работает.** Загрузите `examples/tovary_primer.xlsx`.
   Двигатели с кодом 8501 53 810 0 должны дать «да» по ПП 311.

## Важно перед первым использованием

Файл `spisok.json` в архиве — **неполная заготовка** на 42 позиции.
Пока вы не обновили перечни, второй раздел будет давать ложные «нет»
по многим кодам, включая 8501.

Обязательно сделайте одно из двух:

- нажмите в приложении **«Обновить перечни с alta.ru»** (данные возьмутся
  с сайта напрямую, снимок не понадобится), либо
- соберите полный снимок локально и загрузите его в репозиторий:

  ```bash
  pip install requests beautifulsoup4
  python3 sobrat_spisok.py
  ```

  Скрипт напечатает, сколько позиций найдено в каждом приложении.
  Сверьте выборочно несколько позиций с сайтом, затем загрузите
  получившийся `spisok.json` в репозиторий.

## Что было исправлено в ваших файлах

В `styles.css` ближе к концу висела закрывающая скобка без открывающего
правила:

```css
.mode-toggle select:focus{...}
  *{transition:none!important}
}
```

Потерялась строка `@media (prefers-reduced-motion:reduce){`. Браузер
отбрасывал этот кусок целиком, поэтому отключение анимаций для людей
с чувствительностью к движению не работало. В новом `styles.css`
правило восстановлено.

Разметка первого раздела не изменена ни в одном символе — только
обёрнута в `<div class="tab-panel" id="tab-tablica">`.
