# Таблица товаров для «Заполнителя» (Альта-Софт)

Веб-приложение для специалиста по таможенному оформлению. Из загруженных коммерческих документов (инвойс, упаковочный лист, спецификация) собирает единую таблицу товаров для последующего импорта в программу «Заполнитель» разработчика Альта-Софт.

Работает целиком в браузере: документы **не отправляются ни на какой сервер**, вся обработка выполняется локально. Это важно при работе с коммерческими документами.

## Структура итоговой таблицы

| № | Графа |
|---|-------|
| 1 | Порядковый номер |
| 2 | Наименование товара |
| 3 | Числовой код единицы измерения (ОКЕИ) |
| 4 | Буквенный код единицы измерения (ОКЕИ) |
| 5 | Количество товара |
| 6 | Цена за единицу товара |
| 7 | Общая стоимость товара |
| 8 | Код изделия / артикул |
| 9 | Вес нетто за единицу, кг |
| 10 | Общий вес нетто, кг |
| 11 | Общий вес брутто, кг |
| 12 | № грузового места |

## Правила обработки

- **Поиск данных** ведётся по ключевым словам в заголовках колонок и тексте документов: *наименование, название, модель, артикул, изделие, код изделия, вес нетто, вес брутто, цена, стоимость, количество, ед. изм., грузовое место* и их англоязычные аналоги (*description, article, model, net weight, gross weight, price, amount, qty, place, box* и т. п.).
- **Коды единиц измерения** проставляются по классификатору ОКЕИ (шт → 796 / ШТ, кг → 166 / КГ, компл → 839 / КОМПЛ, пар → 715 / ПАР и др.). Если единица не распознана, графы 3–4 остаются пустыми.
- **Брутто грузового места** распределяется между товарами этого места **пропорционально их весу нетто**, с округлением до трёх знаков после запятой (остаток округления относится на последний товар места, поэтому сумма всегда сходится с брутто места).
- **Повторяющиеся товары** (одинаковое наименование и артикул) объединяются в одну строку; количество, стоимость и веса суммируются. Если товар лежит в нескольких местах, номера мест перечисляются через запятую в графе 12.
- **Расхождения между документами** по количеству, весу или стоимости выделяются **жёлтой заливкой** — и в предпросмотре, и в выгружаемом XLSX (в примечании к ячейке наименования указано, какие документы расходятся и в чём).
- **Недостающие данные** не выдумываются: заполняются только найденные графы, остальные остаются пустыми. Если загружен один файл — таблица строится только по нему.
- Строки «Итого / Всего / Total» из документов в таблицу не попадают.

## Поддерживаемые форматы

| Формат | Качество распознавания |
|--------|------------------------|
| XLSX, XLS, CSV | Высокое — таблицы читаются напрямую |
| DOCX, TXT | Хорошее, если данные оформлены таблицей или строками с разделителями |
| PDF | Текстовые PDF читаются построчно; **сканы без текстового слоя не распознаются** (нужен предварительный OCR) |

## Как запустить на GitHub Pages

1. Создайте репозиторий на GitHub и загрузите в него содержимое этой папки (`index.html`, `styles.css`, `app.js`, `README.md`).
2. Откройте **Settings → Pages**.
3. В разделе **Build and deployment** выберите: Source — *Deploy from a branch*, Branch — `main`, папка — `/ (root)`. Нажмите **Save**.
4. Через одну–две минуты приложение будет доступно по адресу `https://<ваш-логин>.github.io/<имя-репозитория>/`.

Либо через командную строку:

```bash
git init
git add .
git commit -m "Таблица товаров для Заполнителя"
git branch -M main
git remote add origin https://github.com/<ваш-логин>/<имя-репозитория>.git
git push -u origin main
```

Локально приложение можно открыть двойным щелчком по `index.html` — сервер не нужен (требуется доступ в интернет для загрузки библиотек с CDN).

## Проверка работы

В папке `examples/` лежат тестовые файлы — `invoice_primer.xlsx` и `packing_list_primer.xlsx`. Загрузите оба: приложение объединит дубликаты «Фильтр масляный», распределит брутто места № 1 (150 кг) пропорционально нетто насоса и прокладки (139,535 + 10,465) и выделит жёлтым строку «Прокладка резиновая» — по ней количество в инвойсе (100) расходится с упаковочным листом (90).

## Используемые библиотеки (CDN)

- [SheetJS](https://sheetjs.com/) — чтение XLSX/XLS/CSV
- [ExcelJS](https://github.com/exceljs/exceljs) — выгрузка XLSX с заливкой и примечаниями
- [pdf.js](https://mozilla.github.io/pdf.js/) — извлечение текста из PDF
- [mammoth.js](https://github.com/mwilliamson/mammoth.js) — извлечение текста из DOCX

## Ограничения

- Приложение — помощник декларанта, а не замена проверки: перед подачей ДТ сверяйте таблицу с первоисточниками.
- Справочник ОКЕИ включает наиболее употребимые единицы; при необходимости список легко расширить в начале файла `app.js` (массив `OKEI`).
- PDF со сложной версткой таблиц могут распознаваться частично — в этом случае приложение сообщит, что данные в файле не найдены.
